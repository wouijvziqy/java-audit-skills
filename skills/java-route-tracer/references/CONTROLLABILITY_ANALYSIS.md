# 参数可控性判定参考文档

本文档说明参数可控性判定的**核心原则**。LLM 应根据代码语义进行判定，而非匹配固定模式。

---

## 1. 核心判定原则

### 1.1 可控性定义

**参数可控性**：用户输入是否能够到达 Sink 点（敏感操作），以及在什么条件下能够到达。

### 1.2 判定思路

```
1. 追踪参数从 HTTP 入口到 Sink 的完整路径
2. 识别路径上所有对参数的赋值/覆盖操作
3. 判定每个覆盖操作的条件语义
4. 得出可控性结论
```

### 1.3 关键判定项

判定时需要回答：

| 判定项 | 影响 |
|:-----|:-----|
| 参数是否被赋值/覆盖？ | 无覆盖 → 完全可控 |
| 覆盖是否在条件语句内？ | 无条件 → 不可控 |
| 条件的语义是什么？ | 决定可控场景 |
| 用户能否控制条件结果？ | 决定是否可利用 |

---

## 2. 覆盖类型分类

| 覆盖类型 | 判定标准 | 可控性结论 |
|:---------|:---------|:-----------|
| **无覆盖** | 参数直接传递到 Sink，无任何赋值 | ✅ 完全可控 |
| **无条件覆盖** | 赋值操作不在任何条件语句内 | ❌ 不可控 |
| **条件覆盖** | 赋值在条件语句内，需判定条件语义 | ⚠️ 条件可控 |

---

## 3. 条件语义判定原则

### 3.1 判定方法

**不要匹配固定的方法名，而是识别代码的语义意图：**

| 语义类型 | 识别方法 | 可控场景 |
|:---------|:---------|:---------|
| **空值/默认值保护** | 条件判断参数是否为空/null/默认值，为空时设置默认值 | 非空时可控 |
| **白名单验证** | 条件判断参数是否在允许列表中，不在时拒绝或覆盖 | 白名单内可控 |
| **黑名单验证** | 条件判断参数是否在禁止列表中，在时拒绝 | 不在黑名单时可控 |
| **格式/范围校验** | 条件判断参数是否符合特定格式或范围 | 符合格式/范围时可控 |
| **安全检查** | 条件判断参数是否安全（如内网检查、路径检查） | 绕过检查时可控 |

### 3.2 语义识别要点

**空值保护的本质**：当用户未提供值时使用默认值，用户提供值时保留用户值

```
语义：if (用户未提供值) { 使用默认值 }
结论：用户提供值时 → 可控
```

**白名单的本质**：只允许特定值通过，其他值被拒绝或替换

```
语义：if (值不在白名单) { 拒绝或替换 }
结论：值在白名单内时 → 可控（但限于白名单值）
```

**安全检查的本质**：阻止危险值，允许安全值通过

```
语义：if (值危险) { 拒绝或替换 }
结论：值被判定为安全时 → 可控（需检查逻辑是否可绕过）
```

---

## 4. 可控性结论输出

### 4.1 结论类型

| 结论 | 符号 | 含义 | 审计建议 |
|:-----|:-----|:-----|:---------|
| 完全可控 | ✅ | 用户输入无任何限制直达 Sink | 必须审计 Sink 安全性 |
| 条件可控 | ⚠️ | 满足特定条件时用户输入可达 Sink | 需验证条件可否满足 |
| 不可控 | ❌ | 用户输入无法影响 Sink | 可排除该参数的漏洞 |

### 4.2 输出格式

```markdown
| 参数 | Sink类型 | 覆盖类型 | 覆盖条件 | 可控性结论 | 可控场景 |
|:-----|:---------|:---------|:---------|:-----------|:---------|
| {name} | {sink} | {type} | {condition} | {conclusion} | {scenario} |
```

### 4.3 详情输出（条件可控时必须）

```markdown
**参数**: {参数名}
**Sink**: {Sink类型} @ {位置}
**覆盖条件**: {条件表达式}

**条件判定**:
| 条件值 | 执行结果 | 可控性 |
|:-------|:---------|:-------|
| {true时} | {结果} | {可控/不可控} |
| {false时} | {结果} | {可控/不可控} |

**结论**: {可控性结论及可控场景}
```

---

## 5. Sink 类型

| Sink | 说明 |
|:-----|:-----|
| SQL | SQL 语句拼接/执行 |
| COMMAND | 系统命令执行 |
| HTTP | HTTP 请求发起 |
| FILE | 文件读写操作 |
| XML | XML 解析 |
| LDAP | LDAP 查询 |
| EXPRESSION | 表达式解析执行 |
| DESERIALIZE | 反序列化操作 |
| RESPONSE | HTTP 响应输出 |
| PATH | 路径拼接操作 |
