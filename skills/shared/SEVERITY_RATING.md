# 漏洞统一分级标准

所有审计 Skill 共享此分级体系，确保漏洞评级一致性。

---

## 1. 严重等级计算公式

```
严重等级 = f(可达性, 影响范围, 利用复杂度)

Score = R × 0.40 + I × 0.35 + C × 0.25
CVSS 映射 = Score / 3.0 × 10.0
```

---

## 2. 三维评分标准

### 可达性 (Reachability, R)

| 分值 | 级别 | 判定标准 |
|------|------|---------|
| 3 | 高 | 无需认证，HTTP 直接可达 |
| 2 | 中 | 需要普通用户认证 |
| 1 | 低 | 需要管理员权限或内网访问 |
| 0 | 无 | 代码不可达 / 死代码 |

### 影响范围 (Impact, I)

| 分值 | 级别 | 判定标准 |
|------|------|---------|
| 3 | 高 | RCE / 任意文件写入 / 完全数据泄露 / 系统沦陷 |
| 2 | 中 | 敏感数据泄露 / 越权操作 / 部分文件读取 |
| 1 | 低 | 有限信息泄露 / 低影响配置读取 |
| 0 | 无 | 无实际安全影响 |

### 利用复杂度 (Complexity, C) — 反向计分，越容易利用分值越高

| 分值 | 级别 | 判定标准 |
|------|------|---------|
| 3 | 低复杂度 | 单次请求即可利用，无前置条件 |
| 2 | 中复杂度 | 需要构造特殊 payload 或多步操作 |
| 1 | 高复杂度 | 需要特定环境 / 竞态条件 / 链式利用 |
| 0 | 不可利用 | 有效防护，无法绕过 |

## 4. 漏洞编号规范

```
格式: {等级前缀}-{漏洞类型}-{序号}

示例:
  C-SQL-001      严重级 SQL 注入
  H-XXE-002      高危级 XXE
  M-AUTH-003     中危级鉴权缺陷
  L-UPLOAD-001   低危级上传问题
```

### 漏洞类型代码

| 类型代码 | 对应 Skill | 漏洞类型 |
|----------|-----------|---------|
| `SQL` | java-sql-audit | SQL 注入 |
| `XXE` | java-xxe-audit | XXE 外部实体注入 |
| `UPLOAD` | java-file-upload-audit | 文件上传 |
| `FILE` | java-file-read-audit | 任意文件读取 |
| `AUTH` | java-auth-audit | 鉴权绕过 / 越权 |

## 5. 可利用性标注与评级的关系

现有的四级可利用性标注保留，作为评级的输入依据：

| 可利用性 | 对可达性 R 的影响 | 对利用复杂度 C 的影响 |
|----------|-----------------|---------------------|
| ✅ 已确认可利用 | 按实际认证需求打分 | 按实际复杂度打分（已验证可利用） |
| ⚠️ 待验证 | 按实际认证需求打分 | 降 1 级（未验证） |
| ❌ 不可利用 | R = 0 | C = 0 |
| 🔍 环境依赖 | 降 1 级 | 降 1 级 |

## 6. 各漏洞类型的典型评级参考

| 场景 | R | I | C | Score | CVSS | 编号示例 |
|------|---|---|---|-------|------|---------|
| SQL注入 + 无认证 + String拼接 | 3 | 3 | 3 | 3.00 | 10.0 | C-SQL-001 |
| XXE有回显 + 需认证 | 2 | 3 | 3 | 2.60 | 8.7 | H-XXE-001 |
| 文件上传 + 无类型校验 + Web目录 | 3 | 3 | 2 | 2.75 | 9.2 | C-UPLOAD-001 |
| 文件读取 + 有基础路径 + 需认证 | 2 | 2 | 2 | 2.00 | 6.7 | M-FILE-001 |
| 鉴权绕过 + 分号绕过 + 管理接口 | 3 | 2 | 2 | 2.40 | 8.0 | H-AUTH-001 |
| ORDER BY注入 + 环境依赖(Oracle-only) | 1 | 2 | 1 | 1.35 | 4.5 | M-SQL-002 |
| 文件读取 + 白名单校验不严格 | 2 | 1 | 1 | 1.40 | 4.7 | M-FILE-002 |
| Session超时过长 | 1 | 1 | 1 | 1.00 | 3.3 | L-AUTH-002 |
